### 代码实现指南

编写代码时需遵循以下规则：



* 尽可能使用「尽早返回」（early returns），提升代码可读性。

* 样式统一使用 Tailwind 类名，禁止使用原生 CSS 或 `<style>` 标签。

* 条件类名优先使用 `class:` 语法，而非三元运算符。

* 变量、函数 / 常量命名需具有描述性。事件函数统一以 `handle` 为前缀命名（如 `onClick` 对应 `handleClick`，`onKeyDown` 对应 `handleKeyDown`）。

* 为元素实现无障碍功能（Accessibility）。例如，可交互元素需添加 `tabindex="0"`、`aria-label` 属性，同时实现 `onClick` 和 `onKeyDown` 等事件处理。

* 函数优先使用 `const` 定义（如 `const toggle = () => {}`），可能的情况下补充类型定义。
